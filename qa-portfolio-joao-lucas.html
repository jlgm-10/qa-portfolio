BUG-001 · LOGIN
Sem limite de tentativas de login — vulnerabilidade de segurança
🔴 CRÍTICO ABERTO Segurança
▼
Descrição
O sistema não limita o número de tentativas de login falhadas. É possível tentar infinitas combinações de password sem ser bloqueado, banido ou apresentar CAPTCHA. Isso expõe a aplicação a ataques de força bruta.
Impacto
Um atacante pode automatizar milhares de tentativas de login por segundo, comprometendo contas de utilizadores. Em e-commerce, isso representa risco directo de fraude e acesso não autorizado a dados de pagamento.
Passos para Reproduzir
  1. Aceder a demo.opencart.com/index.php?route=account/login
  2. Inserir email válido com password errada
  3. Clicar em "Login" repetidamente (10+ vezes)
  4. Observar: nenhum bloqueio, CAPTCHA ou aviso é apresentado
Resultado Esperado
Após 3–5 tentativas falhadas, o sistema deve bloquear temporariamente o acesso ou exigir CAPTCHA
Resultado Obtido
Sistema aceita tentativas ilimitadas sem qualquer mecanismo de protecção
Sugestão de Correção
Implementar rate limiting por IP (máx. 5 tentativas/15 min) e CAPTCHA após 3 falhas consecutivas. Considerar autenticação de dois factores (2FA).
BUG-002 · PESQUISA
Pesquisa com campo vazio retorna mensagem incorrecta
🟠 ALTO ABERTO UX / Funcionalidade
▼
Descrição
Ao pesquisar com campo vazio, o sistema exibe "There is no product that matches the search criteria" em vez de uma mensagem orientativa como "Por favor introduza um termo de pesquisa". O utilizador fica confuso, pensando que a loja não tem produtos.
Passos para Reproduzir
  1. Aceder a demo.opencart.com
  2. Deixar a barra de pesquisa vazia
  3. Clicar na lupa ou premir Enter
  4. Observar a mensagem exibida
Resultado Esperado
Mensagem: "Por favor introduza um termo de pesquisa" ou todos os produtos são listados
Resultado Obtido
Mensagem: "There is no product that matches the search criteria" — enganosa para o utilizador
BUG-003 · CARRINHO
Mensagem de confirmação de adição ao carrinho desaparece sem interacção
🟡 MÉDIO ABERTO UX
▼
Descrição
Após adicionar um produto ao carrinho, a mensagem de confirmação verde desaparece automaticamente após poucos segundos sem que o utilizador possa dispensá-la manualmente ou clicar em "Ver carrinho" antes de desaparecer.
Passos para Reproduzir
  1. Pesquisar qualquer produto (ex: "MacBook")
  2. Clicar em "Add to Cart"
  3. Observar a notificação verde no topo
  4. Não interagir — a notificação desaparece em ~3 segundos
Resultado Esperado
Notificação permanece visível até o utilizador a dispensar ou ter tempo suficiente para ler e agir
Resultado Obtido
Notificação desaparece automaticamente em ~3 segundos, podendo o utilizador não a ter lido
BUG-004 · LOGIN
Mensagem de erro de login não diferencia email inválido de password errada
🟡 MÉDIO ABERTO UX
▼
Descrição
A mesma mensagem de erro é exibida tanto para email inexistente como para password incorrecta. Embora isto seja uma boa prática de segurança, a mensagem actual é técnica e pouco amigável para utilizadores comuns.
Resultado Esperado
Mensagem clara e amigável: "Email ou password incorrectos. Por favor tente novamente."
Resultado Obtido
"Warning: No match for E-Mail Address and/or Password." — linguagem técnica e confusa
BUG-005 · UI
Número de telefone de contacto é um placeholder não funcional
🔵 BAIXO ABERTO Conteúdo
▼
Descrição
O número de telefone exibido no topo da página ("123456789") é claramente um dado de teste/placeholder e não um contacto real. Em ambiente de produção, isto indicaria um dado de configuração por preencher.
Resultado Esperado
Número de contacto real ou campo removido se não aplicável
Resultado Obtido
Exibido "📞 123456789" — número sequencial claramente fictício
